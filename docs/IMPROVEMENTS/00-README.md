# ReadAloud — Improvement Proposals

This folder collects improvement ideas for ReadAloud across four dimensions — new features, user interests & engagement, code structure, and UI & voice. They were generated by a fan-out of specialized review agents, each auditing one dimension against the live codebase (real files, real line numbers) and against Ruby's hard rules. **Everything here is a SUGGESTION, not a decision.** The intent is a skimmable menu Ruby can triage — keep, defer, or drop each item — and hand the survivors to an implementer. Nothing has been built; no code was changed.

## Cross-dimension overlaps (don't double-count)

Several items surface in more than one doc because they need work across layers — treat these as **one piece of work**, not several:

- **`.dictionaryLookup(term:)` modifier** — appears in both Code (03) and UI (04); it's a single extraction.
- **Shared translate helper / `TranslationResolver`** — the "extract `Meaning` ×3" cleanup in Code (03) is the *same* work as Engagement's (02) prerequisite for Quick-Scan digest, phrasebook, and subtitle surfaces. Do it once; several features then ride on it.
- **`SavedWord` → `Annotation` port** — the schema work in Code (03) is what makes UI's (04) "collapse Item-notes segment" and "reconcile Saved vs Notebook tabs" disappear. One port, three payoffs.
- **Audio-capture loop** — the Features (01) capability and the Engagement (02) "Media learner" bet are the same in-flight `AUDIO_LEARNING_DESIGN.md` direction; `SentencePlayer` protocol (03) is its code-side enabler.
- **`ReplayButton` / `SRSStatsSection` extraction** — Code (03) reuse move that also closes UI's (04) Reduce-Motion gap on the bounce; the extraction and the a11y fix land together.
- **Widgets / App Intents / extensions** — every item in Features §6 shares one prerequisite: move the SwiftData store to an App Group. Pay it once before picking any of them.
- **Mastery / summary "what's next" copy** — Engagement's (02) delight moments and UI's (04) session-summary rewrite touch the same `ReviewSessionView` summary surface; coordinate so the copy and the trigger match.

## Remember Ruby's constraints (every pick must honor them)

**Fully offline, Apple frameworks only, no networking, no third-party deps** · **reuse-first — name the existing model/service/component it extends, keep the app concise** · **clear folders** (`Features/<Screen>`, `Services`, `Models`, `Shared/{Components,Styles,Extensions}`, `Tools`, `docs`; pure logic → `Packages/`) · **energetic but NOT gamified** — no streaks, XP, points, badges, or leaderboards; build on paper & ink (#36) + semantic palette (#39) + DESIGN_GUIDELINES.md · **warm, grown-up tone** for adult learners. French → English primary.
